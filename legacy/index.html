A plain-English crash course

Never touched cryptography, bitcoin, or a puzzle like this? Perfect. Read these six cards and you’ll understand the entire machine. Every technical word below is explained with an everyday analogy — and you can do all of it right on this page.

🏆What is this, and what’s the prize?

It’s a treasure hunt where the treasure is real money. In 2017 the creators hid 5 bitcoins behind a chain of puzzles and dared the world to solve it. Whoever cracks the final puzzle recovers a secret “master password” that lets them take the coins.

The hoard sits at a public bitcoin address that anyone can watch. It has never been emptied — proof that, after years of attempts, nobody has solved it. (Bitcoin’s built-in “halvings” have reduced the main pile to ~1.25 BTC over time, still a small fortune.)

Analogy: a locked treasure chest sitting in a glass case in the town square. Everyone can see it’s still locked; the hunt is to find the one key that opens it.

₿Bitcoin, addresses & private keys

Bitcoin is money that lives on the internet with no bank in charge. Coins sit at an address — a public string like 1GSMG1…prBe that works like an account number anyone can see and send to.

To spend from an address you need its private key: a secret string only the owner knows. Whoever holds the private key controls the coins. This entire puzzle is a machine for computing one private key.

Analogy: the address is your visible mailbox number; the private key is the only key to open it. Find the key, own what’s inside.

🧮What is a “hash” (SHA-256)?

A hash is a one-way fingerprint machine for text. Feed it anything, it returns a fixed 64-character code. Same input → same code, every time. Change a single letter → a completely different code. And you cannot run it backwards to recover the input.

This puzzle uses SHA-256 (a famous, standard hash) as the glue between doors: each riddle’s answer is fed through SHA-256, and the resulting code becomes the key that opens the next locked box.

Analogy: a meat grinder. Sausage comes out the same every time for the same meat — but you can never turn the sausage back into the cow.

🔐What is encryption (AES)?

Encryption scrambles a message into gibberish that only the exact right key can unscramble. The scrambled blob is safe to show the whole world; without the key it’s meaningless.

Each door here is a page locked in an AES box (specifically AES-256-CBC). The key to each box is the SHA-256 code from the step before. Get the answer wrong by one character and the box yields pure garbage — there’s no “close enough”.

Analogy: a strongbox in plain sight. Anyone can rattle it; only the precise key opens it. The right key here is a hash.

🗝What is a “cipher”?

Long before computers, people hid messages with ciphers — simple rules like “shift every letter forward by three” or “swap letters using a secret table.” They’re puzzles, not unbreakable locks.

The puzzle’s later doors stack a few classic ciphers (with names like EBCDIC, Beaufort, and VIC) on top of the modern AES box, as extra riddles. Each is just a different recipe for turning scrambled symbols back into words.

Analogy: a decoder ring from a cereal box — turn the dial to the secret setting and the gibberish becomes a sentence.

⚙️How the whole machine works

Every door has the same shape: solve a riddle → get an answer in words → run it through SHA-256 → use that code to open the next AES box → inside is the next riddle. Repeat eight times.

The eighth door opens onto the final stage (SalPhaseIon + Cosmic Duality), where the last box’s key would reveal the private key to the 5 BTC. Eight doors are open and fully explained below; the last is where everyone is stuck.

In plain English: how a picture becomes a web address+

Computers store everything as binary — long runs of two symbols, 0 and 1. This image is binary in disguise: each little square is one bit. Black or blue = 1, white or yellow = 0.

Reading order matters. You start at the top-left corner and move in a spiral, curling inward, going down the left side first. That gives you a long ribbon of 196 ones and zeros.

Computers group bits into bytes of eight, and each byte is the code for one character (this is called ASCII). So you chop the ribbon into groups of 8. The first group is 01100111, which is the number 103, which is the letter g.

Keep going and the squares spell a web address: gsmg.io/theseedisplanted — the door to Phase 1. The leftover 4 squares at the end are just padding (zeros).

The handful of blue and yellow squares looked like they might hide a second secret (an old hint teased “Yellow has a number and so does Blue”). They don’t: they simply mark the last bit of each letter. Mystery solved, no extra message.

In plain English: three antique codes hidden in one box+

Open this box with Phase 3’s password. But this time you don’t get plain words — you get more puzzles, written in old secret codes. Three of them, peeled in order:

① EBCDIC — a block of weird line-drawing symbols is actually text written in an alphabet IBM used on 1960s mainframe computers. The clue “one for one, four for one” names the exact version, number 1141. Translate it back to normal letters.

② Beaufort cipher — those letters are still scrambled by a letter-swapping method called Beaufort. Its secret keyword is the Matrix line THEMATRIXHASYOU. Unscramble, and out comes a dramatic speech by “The Architect” — which is secretly the creator’s instruction sheet for the endgame.

③ VIC cipher — separately, a long run of digits on the page is a Cold-War spy code called VIC. Decoded, it reads a key line: the private keys belong to “half and better half”.

The Architect’s speech and that VIC line are the breadcrumbs for the final stage — but the door itself is already open here.

09 — THE HIVE

Collaborative leaderboard — let’s beat it together

The endgame is a search problem: somewhere there is a recipe that opens Cosmic Duality, and the only way to find it is to try candidates. This turns that search into a shared, deduplicated, self-verifying effort — so no two people waste time on the same idea, the ruled-out space is recorded forever, and a winning attempt is impossible to miss.

How this actually helps beat the puzzle (and why the scores are trustworthy)+

Every time you try a passphrase in the workbench above, this page runs the real AES decryption and records the true cryptographic result — unlocked (valid padding) or fail. Because the encrypted blobs are public and the maths is deterministic, anyone can reproduce any attempt, and a GitHub robot re-runs every submitted attempt to confirm it before it counts. So “reliable” here isn’t a popularity vote — it’s cryptographically checked truth.

Attempts are deduplicated by an exact fingerprint of (which blob + the precise recipe + whether it was hashed first). Testing the same recipe twice doesn’t inflate anyone’s score; only genuinely new ground counts. That’s what makes the collective search efficient: the leaderboard is a live map of what’s been covered.

Win condition: the first person whose attempt produces a valid decryption of the Cosmic Duality blob has effectively solved the puzzle — the page detects it automatically and lights up. Everything else is about covering the search space until then.

11 — SOURCES & A WARNING

Sources, tools & the “solutions” problem

▲ On the “I solved it” posts

The puzzle’s GitHub issues host a steady stream of confidently-worded “solutions” — XOR-master-key reports, “the private key is Satoshi’s genesis address”, reconstructed forensic PDFs. None is real. They share a tell: a few correct facts wrapped around invented machinery that traces to nothing the creators wrote. The prize address is unspent; dbbi, faed, the inner blob and Cosmic Duality remain genuinely open. Treat any “solution” without a spendable key with deep skepticism — and you can disprove most of them in seconds with the workbench above.
